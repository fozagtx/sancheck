# Plugin Skills

sancheck includes a `url-gate` skill that instructs the Codex agent how to use the scanner.

## url-gate Skill

Location: `plugins/sancheck/skills/url-gate/SKILL.md`

### Purpose

Gate URLs with the bundled sancheck scanner before an agent opens, fetches, summarizes, or acts on untrusted links.

### When to Use

Use this skill whenever a task includes external links or asks you to:

- Open a URL
- Fetch a URL
- Summarize a URL
- Inspect a URL
- Crawl a URL
- Act on an untrusted URL

### Workflow

1. **Resolve the plugin root**: The directory two levels above `SKILL.md`.
2. **Send untrusted text**: Pipe the text, prompt, JSON payload, or URL list to `scripts/sancheck-gate`.
3. **Check exit code**: Continue only when exit `0` and `"allowed": true`.
4. **Handle blocks**: If exit `2`, do not open blocked URLs. Report them.
5. **Handle failures**: If exit `1`, treat as failed closed and ask for user direction.

### Command

```bash
printf '%s\n' "$UNTRUSTED_TEXT_OR_JSON" | "$PLUGIN_ROOT/scripts/sancheck-gate"
```

### Direct URL Form

```bash
"$PLUGIN_ROOT/scripts/sancheck-gate" https://example.com
```

### Output

The gate emits JSON with:

- `decision`: `allow` or `block`
- `allowed`: `true` or `false`
- `blocked_urls`: List of blocked URLs
- `reports`: Full scan reports for each URL

### Provider Keys

The scanner uses live DNS, TLS, HTTP, RDAP, and content checks by default. Optional provider checks run only when these environment variables are set:

```bash
GOOGLE_SAFE_BROWSING_API_KEY
VIRUSTOTAL_API_KEY
PHISHTANK_APP_KEY
```

Missing provider keys are reported as `skipped`, not as clean results.

## Creating Custom Skills

You can create custom skills by adding new directories under `plugins/sancheck/skills/`. Each skill should have a `SKILL.md` file with:

1. **Frontmatter**: `name` and `description` fields.
2. **Purpose**: What the skill does.
3. **When to Use**: When the agent should invoke the skill.
4. **Workflow**: Step-by-step instructions.
5. **Command**: The command to run.
6. **Output**: Expected output format.

### Example: Custom Skill

```markdown
---
name: my-custom-skill
description: Custom skill for my use case.
---

# My Custom Skill

## Purpose

...

## When to Use

...

## Workflow

1. ...
2. ...

## Command

...

## Output

...
```
