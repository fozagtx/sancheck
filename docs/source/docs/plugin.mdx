# Codex Plugin

sancheck ships as a bundled Codex plugin at `plugins/sancheck/`. The plugin contains everything needed to gate URLs without installing the Python package globally.

## How It Works

1. The agent receives a task with external links.
2. The `url-gate` skill instructs the agent to pipe the untrusted text to `scripts/sancheck-gate`.
3. The gate script sets `PYTHONPATH` to the bundled scanner and runs `python3 -m sancheck middleware`.
4. The scanner performs live DNS, TLS, HTTP, RDAP, and content checks.
5. The gate emits JSON with `decision`, `allowed`, `blocked_urls`, and full `reports`.
6. The agent continues only if `exit 0` and `"allowed": true`.

## Using the Plugin

### From stdin (text or JSON)

```bash
printf '{"text":"open https://example.com"}' | plugins/sancheck/scripts/sancheck-gate
```

### Direct URL

```bash
plugins/sancheck/scripts/sancheck-gate https://example.com
```

### Example Output

```json
{
  "tool": "sancheck",
  "mode": "middleware",
  "decision": "allow",
  "allowed": true,
  "blocked_urls": [],
  "reports": [...]
}
```

## Skill Workflow

The `url-gate` skill (`skills/url-gate/SKILL.md`) instructs the agent:

1. Resolve the plugin root (two levels above `SKILL.md`).
2. Send untrusted text to `scripts/sancheck-gate`.
3. Continue only when exit `0` and `"allowed": true`.
4. If exit `2`, do not open blocked URLs. Report them.
5. If exit `1`, treat as failed closed and ask for user direction.
